Advanced options (time-window scoping, severity, analysis focus areas)

Detecting this machine's CPU core count…

"Auto" detects CPU count and available memory and uses multiple processes to scan large bundles' files concurrently — small bundles stay single-process automatically since the speedup wouldn't be worth the overhead. Picking a number above this machine's own CPU core count ("oversubscribing") can still help on bundles dominated by many small files (I/O-wait-bound — e.g. under real-time antivirus/EDR scanning overhead) but rarely helps, and can slightly hurt, on bundles dominated by a few huge files (CPU-bound regex scanning) — try it both ways if you're unsure which shape your bundle is. Force "1" to compare timing or rule out a parallelism-related issue.

Only chronological log-like files are gated by a time window — configuration dumps and other point-in-time snapshots are always fully scanned regardless.
